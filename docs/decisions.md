# Decisions

## Initial decisions

1. Проект реализуется как Telegram-бот на Python.
2. Используется PostgreSQL как основная база данных.
3. Используется одна база данных на всю сеть кафе.
4. Разделение данных между кафе реализуется через cafe_id.
5. Работа с БД будет через SQLAlchemy ORM.
6. Изменения схемы БД будут делаться через Alembic.
7. В MVP анкета содержит 4 boolean-поля для ответов.
8. Комментарий гостя хранится как необязательный текст.
9. Основная архитектура: `handlers -> services -> repositories -> db.`
10. AI-анализ комментариев является дополнительной функцией, не критичной для ядра системы.

## Логика подсчета баллов в опросе

Каждый опрос содержит 4 логических ответа.

- Да = 1
- Нет = 0

Балл опроса рассчитывается как сумма всех положительных ответов.

Возможные значения:
- 4/4 = 100%
- 3/4 = 75%
- 2/4 = 50%
- 1/4 = 25%
- 0/4 = 0%
